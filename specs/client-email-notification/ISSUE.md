# [Feature] S3データ不着検知時にクライアントへメールで通知する手段がない

> GitHub Issue #3 より転載（2026-05-25）

## 背景・問題

S3 へのデータ不着は現在検知できているが、通知先は社内 Discord のみで、
クライアントへの直接通知手段がない。
データ欠損が発生してもクライアントが自力で気づくまで時間がかかる。

## ユーザーストーリー

データ送信者（クライアント）として、
S3 に期待するデータが届いていない際にメールで通知を受け取りたい。
なぜなら、データ欠損を早期に検知し、迅速に対処できるようにしたいから。

## 受け入れ条件

- [x] メール送信手段を選定する（SES / SNS Email / GAS 等の比較・決定）→ RESEARCH-001.md 参照
- [x] エラー種別ごとに通知チャネル（Discord / メール等）を `error-routing.yml` で制御できる → SPEC REQ-005 / DESIGN 設計済み
- [x] メールアドレスグループを `email.yaml` で定義し、エラー種別ごとに送信グループを指定できる → SPEC REQ-006 / DESIGN 設計済み
- [ ] S3 データ不着の検知時に、指定グループのクライアントへメール通知が送られる → TASK-005, TASK-007 実装待ち
- [ ] 通知メールに船舶名と未着確認時刻（JST）が含まれる → TASK-006 実装待ち
- [ ] メールの HTML テンプレートを実装する → TASK-006 実装待ち

## スコープ外（今回は扱わない）

- 重複通知の抑制ロジック
- Discord 通知の廃止・変更
- メール開封率・クリック率のトラッキング
- alarm → error_id のマッピングの設定ファイル化（コードに直書きで管理）
- SES バウンス・苦情通知の自動処理
