## 各サービスの概要

### Amazon SES（Simple Email Service）

AWS が提供するメール送信専用サービス。Lambda から boto3 で直接呼び出せる。
送信元ドメインの DNS 設定（DKIM）とサンドボックス解除申請が初期に必要。

### Amazon SNS（Simple Notification Service）Email サブスクリプション

AWS の pub/sub メッセージングサービス。本来はシステム間の通知用で、メール配信は補助機能に過ぎない。
**平文のみ対応のため、本要件（HTML テンプレート）を満たせず除外。**

### Google Apps Script（GAS）

Google のローコードスクリプト実行環境。Gmail/Google Workspace アカウントに紐づいた Web API エンドポイントを作り、Lambda から HTTP POST でメール送信を依頼する構成。
**Google Workspace 未使用のため Gmail 個人上限（100通/日）となり、また `@scrumsign.com` からの送信が不可のため除外。**

### Resend

2023年設立の開発者向けトランザクションメール API サービス。AWS Lambda との公式インテグレーションガイドあり。
サンドボックス制限なし・即日送信可能が特徴。AWS 外部サービスのため API キー管理が必要。

## クライアント（受信者）側の負担 — 詳細

### Amazon SNS Email：クライアントに最大の負担

SNS は受信者側の手間が最も大きく、本番運用で深刻な問題を引き起こす可能性がある。

#### 1. 確認メールのクリックが必須

新しいメールアドレスを登録するたびに、必ず以下のフローが発生する。

1. AWS から `no-reply@sns.amazonaws.com` を送信元とする確認メールが届く
2. 本文は英語の平文システムメッセージ（ビジネスメールとは程遠い見た目）で、クライアントがフィッシングと誤認するリスクがある
3. クライアントが「Confirm subscription」リンクをクリックして初めて受信開始
4. **トークンの有効期限は 48時間**。クリックされなければ登録は削除され、送信者側が再登録する必要がある

これは仕様であり、回避手段はない。

#### 2. セキュリティスキャナによる意図しない購読解除

企業のメールセキュリティ製品（Microsoft Defender for Office 365、Proofpoint、Mimecast 等）は、受信メール内の全 URL を自動的に先読み（プリフェッチ）してウイルスチェックを行う。

SNS メールには購読解除 URL が含まれており、**セキュリティスキャナがこの URL を先読みした瞬間に AWS は「解除操作」として処理してしまう**。クライアントは何もしていないのに、アラートメールが届かなくなる。

- 発生環境：Microsoft 365 / Exchange を使う企業（日本の大企業・官公庁の多くが該当）
- AWS re:Post にも「Microsoft 365 で SNS の確認メールが届かない」報告が多数
- 回避策：`AuthenticateOnUnsubscribe=true` を設定するが、完全ではなく運用が複雑になる

#### 3. メールアドレス変更のたびに再クリックが必要

クライアントのメールアドレスが変わるたびに、旧アドレスの削除 → 新アドレスの登録 → **クライアントに確認クリックを依頼** というフローが必要になる。

#### まとめ

SNS Email は「社内の技術者が自分で確認クリックをする」内部通知には使えるが、外部クライアントへの通知では手間とリスクが高すぎる。

---

### Amazon SES：クライアントの負担ゼロ

本番アクセス（サンドボックス解除）後は、クライアントは何もしなくてよい。

- 登録の手続きは送信者側のみ（メールアドレスをコードまたは設定に追加するだけ）
- メールアドレス変更も送信者側のみ対応（クライアントへの案内は不要）
- メールの内容・送信者名・件名はすべて送信者が自由に制御できる

**届かないリスクについて**: SES は共有 IP プールを使うため、他の送信者の評判の影響を受ける。独自ドメイン（`@scrumsign.com`）+ DKIM/SPF/DMARC を設定することでリスクを大幅に低減できる。独立調査（2024-2025年）による受信トレイ到達率は約 75%（共有 IP の場合）。DKIM 設定が適切であれば企業の Outlook でも問題なく受信できる。

---

### Resend：クライアントの負担ゼロ

SES 同様、クライアントは何もしなくてよい。

- 受信開始前の操作不要
- メールアドレス変更も送信者側のみ対応
- メールの内容・送信者名は送信者が完全制御

---

## 日本の法規制（特定電子メール法）

### 結論：本要件には適用されない

特定電子メール法が定める「特定電子メール」とは、**「自己または他人の営業につき広告または宣伝を行うための電子メール」**のことを指す。

S3データ不着を検知してクライアントに通知するアラートメールは：

- 広告・宣伝目的ではない
- 契約に基づくサービス運用上の通知（取引関係通知）
- 広告サイトへの誘導 URL を含まない

これらの条件から**適用外**となる。オプトイン（事前同意）の取得義務はない。

また仮に適用対象とみなされる場合でも、契約関係のある既存クライアントへの送信は例外規定が適用される。

**実務上の推奨**: 法的義務はないが、サービス開始時の案内でアラートメールが届く旨を伝え、問い合わせ窓口（配信停止の連絡先）を明示しておくと丁寧。

## Amazon SES の詳細

### コスト

- **$0.10 / 1,000通**（$0.0001/通）
- **無料枠**: 3,000通/月（初年、アカウント作成から 12ヶ月）
- 今回の想定量（最大 6通/日 ≒ 180通/月）は無料枠内

### サンドボックス制限

デフォルト状態では外部メールアドレスへの送信不可。**本番アクセス申請（AWS Support チケット）が必要。**

- 無料サポートプランの場合、承認まで 24〜96時間（SLA なし）
- サンドボックス解除前は検証済みアドレスにしか送れない

### `@scrumsign.com` から送信するための手順

1. SES コンソールで `scrumsign.com` ドメインを作成
2. DNS に 3 件の CNAME レコードを追加（Easy DKIM 用）
3. DNS 伝播を待つ（最大 72時間、通常は数時間）
4. サンドボックス解除申請を送信（AWS Support）
5. 承認後、任意の `@scrumsign.com` アドレスから送信可能

### Lambda 統合（Python boto3）

```python
import boto3

ses = boto3.client('ses', region_name='ap-northeast-1')

ses.send_email(
    Source='alerts@scrumsign.com',
    Destination={'ToAddresses': ['client@example.com']},
    Message={
        'Subject': {'Data': 'S3 データ不着アラート', 'Charset': 'UTF-8'},
        'Body': {
            'Html': {'Data': '<h1>アラート</h1><p>...</p>', 'Charset': 'UTF-8'},
            'Text': {'Data': 'アラート: S3 データ不着...', 'Charset': 'UTF-8'},
        },
    },
)
```

### IAM ポリシー（Lambda 実行ロールに追加）

```json
{
  "Effect": "Allow",
  "Action": ["ses:SendEmail", "ses:SendRawEmail"],
  "Resource": "arn:aws:ses:<region>:<account-id>:identity/scrumsign.com"
}
```

## Resend の詳細

### コスト

- **無料枠**: 3,000通/月（100通/日上限）
- 有料プラン: $20/月 で 50,000通（$0.40/1,000通）
- 今回の想定量（最大 180通/月）は無料枠内

### サンドボックスなし

ドメイン認証が完了すれば即日任意アドレスへ送信可能。AWS サポートへの申請は不要。

### `@scrumsign.com` から送信するための手順

1. Resend にアカウント作成
2. `scrumsign.com` ドメインを追加（DNS に SPF/DKIM レコードを追加）
3. DNS 伝播後、即日 `@scrumsign.com` から送信可能
4. API キーを発行し、Lambda の環境変数（または Secrets Manager）に設定

### Lambda 統合（Python）

```python
import urllib.request
import json

def send_email(to: str, subject: str, html: str) -> None:
    payload = json.dumps({
        'from': 'alerts@scrumsign.com',
        'to': [to],
        'subject': subject,
        'html': html,
    }).encode('utf-8')

    req = urllib.request.Request(
        'https://api.resend.com/emails',
        data=payload,
        headers={
            'Authorization': f'Bearer {RESEND_API_KEY}',
            'Content-Type': 'application/json',
        },
    )
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read())
```

## タイムライン比較

| | **Amazon SES** | **Resend** |
|---|---|---|
| DNS レコード追加（`scrumsign.com`） | 必要・最大 72時間伝播 | 必要・最大 72時間伝播 |
| サンドボックス解除申請 | 必要（24〜96時間） | **不要** |
| 外部送信開始まで | 2〜5日（DNS + 申請の直列） | DNS 伝播後すぐ |
| DNS なしの暫定稼働 | 不可 | 共有ドメイン（`onboarding@resend.dev`）で暫定可 |
